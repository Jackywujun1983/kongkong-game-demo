# 项目开发规范

本工作区内的所有开发任务默认遵守以下规范文档：

- `docs/规范/Python编码规范.md`
- `docs/规范/前端主流开发框架规范.md`
- `docs/规范/测试规范.md`

## 执行要求

- 编写、修改、评审 Python 代码时，必须遵守 `docs/规范/Python编码规范.md`。
- 编写、修改、评审前端代码时，必须遵守 `docs/规范/前端主流开发框架规范.md`。
- 编写、修改、评审测试代码或测试方案时，必须遵守 `docs/规范/测试规范.md`。
- 如果具体项目已有更细的局部规范，应在不冲突的前提下同时遵守；如发生冲突，优先确认项目内更具体的约定。
- 后续新增代码、重构、测试、文档和工程配置，应尽量与上述规范保持一致。
