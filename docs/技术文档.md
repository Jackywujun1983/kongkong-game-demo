# 空空如也GameHub 技术设计文档

更新日期：2026-06-14

## 1. 项目概览

空空如也GameHub 是一个前后端分离源码组织的游戏信息检索网站。当前本地运行时使用 Python 标准库 HTTP 服务同时托管静态前端页面和 `/api` 接口，便于直接启动和验证。

当前版本只保留游戏检索、类型筛选、详情、分页、静态导出和广告位预留接口。账户注册、登录权限和论坛模块已经下线。

## 4. 架构说明

```text
浏览器
  ├─ 本地运行：访问 Python HTTP 服务
  │   ├─ 托管 frontend/preview.html、detail.html 和静态资源
  │   └─ 提供 /api REST 接口
  │       └─ SQLite
  └─ GitHub Pages：访问 frontend 静态文件
      └─ API 不可用时读取 frontend/public/game-data.js
```

当前源码保持前后端目录分离：

- `backend/` 负责 API、数据库、仓储和服务启动。
- `frontend/` 负责静态页面、样式、默认封面和 React 工程源码。
- `docs/` 负责需求、技术、数据库、部署、规范和页面布局文档。
- `tools/` 负责数据导入、静态数据导出和发布辅助脚本。

## 7. API 接口

本地基础地址：

```text
http://127.0.0.1:8000/api
```

### 7.1 健康检查

```http
GET /api/health
```

### 7.2 类型列表

```http
GET /api/categories
```

返回规则：

- 只返回 `is_visible = 1` 的类型。
- 按 `game_count DESC, name ASC` 排序。

### 7.3 游戏检索

```http
GET /api/games?query=银河文明&category=策略&page=1&page_size=48
```

参数：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `query` | string | 否 | 搜索关键词 |
| `category` | string 或重复参数 | 否 | 类型 slug 或类型名称，支持逗号分隔或重复传参 |
| `page` | number | 否 | 页码，最小值为 `1` |
| `page_size` | number | 否 | 每页数量，后端最大限制为 `50` |

搜索规则：

- `query` 对 `games.title`、`games.summary`、`games.tags`、`games.studio` 做大小写不敏感 `LIKE`。
- `category` 使用 `game_categories` 关系表匹配。
- 类型匹配精确比较 `categories.slug` 或 `categories.name`。
- 类型匹配不使用 `LIKE`。
- 多个类型之间是 OR 关系。
- 关键词和类型同时存在时，两组条件之间是 AND 关系。

核心响应字段：

| 字段 | 说明 |
| --- | --- |
| `items` | 当前页游戏列表 |
| `page` | 当前页码 |
| `page_size` | 当前页大小 |
| `total` | 当前筛选条件下的总数 |
| `items[].categories` | 游戏拥有的全部类型 |
| `items[].download_url` | 下载地址 |
| `items[].size` | 游戏大小 |

### 7.4 游戏详情

```http
GET /api/games/{slug}
```

返回单个游戏完整信息，并包含 `details` 和全部类型列表。

### 7.5 广告位预留

```http
GET /api/ads?placement=home_top
```

当前前端不展示广告位，该接口仅作为后续扩展预留。

### 7.6 已移除接口

以下接口不再提供：

- `POST /api/auth/register`
- `POST /api/auth/login`
- `GET /api/me`
- `GET /api/forum/posts`
- `POST /api/forum/posts`
- `GET /api/forum/posts/{id}`
- `POST /api/forum/posts/{id}/comments`

## 8. 前端页面

| 页面 | 文件 | 说明 |
| --- | --- | --- |
| 根入口 | `frontend/index.html` | GitHub Pages 根路径跳转到 `preview.html` |
| 首页 | `frontend/preview.html` | 游戏库、搜索、类型筛选、焦点位、统计区、分页 |
| 详情页 | `frontend/detail.html` | 游戏详情、下载、同分类推荐 |
| 样式 | `frontend/public/site.css` | 当前静态页面主样式 |
| 兜底数据 | `frontend/public/game-data.js` | API 不可用时的静态数据 |

已删除：

- `frontend/auth.html`
- `frontend/forum.html`

当前前端 UI 要点：

- 整体为浅色纸面风格、粗描边、低饱和色彩。
- 首页和详情页顶部不展示“游戏库”导航按钮。
- 详情页正常状态不展示“返回游戏库”和面包屑导航。
- 详情页顶部使用 `.detail-topbar` 品牌栏，左侧品牌可点击回首页，右侧装饰轨仅用于视觉平衡。
- 首页 H1-1 焦点位展示封面、标题、简介摘要和查看详情按钮。
- 首页 H1-2 统计区展示收录游戏、当前焦点游戏类型、当前结果最大年份、当前焦点游戏大小。
- 首页 H4 普通游戏卡片只展示封面、游戏名、类型信息行和查看详情按钮。
- 首页 H4 普通游戏卡片不展示年份和游戏大小。
- 静态页 CSS 和默认封面引用带资源版本号，用于降低 GitHub Pages/CDN 缓存影响。

## 9. 数据导入与静态导出

### 9.1 CSV 导入

导入脚本：

```text
tools/import_fufu_quark_games.py
```

作用：

- 读取 `data/fufu_quark_only.csv`。
- 按 CSV 的游戏类型字段拆分类型。
- 写入或更新 `games`。
- 写入 `download_url`、`size`、`cover_url`、`summary`。
- 同步 `game_categories` 多类型关系。
- 清理低频类型展示状态并刷新 `categories.game_count`。

### 9.2 静态数据导出

导出脚本：

```text
tools/export_static_game_data.py
```

作用：

- 读取 `backend/gamehub.sqlite3`。
- 导出可见类型。
- 导出全部游戏、全部类型关系、详情、下载地址、大小、平台、标签和封面。
- 写入 `frontend/public/game-data.js`。
- 同步复制数据库引用到的真实游戏封面到 `frontend/pic/`。

GitHub Pages 工作流会在部署前自动执行该脚本。

## 11. 测试

后端测试：

```bash
cd backend
python -m unittest discover -s tests
```

当前测试覆盖：

- 游戏搜索。
- 游戏详情。
- 多类型 OR 筛选。
- 分类和广告种子数据。
- 已移除账户/论坛表不再初始化。
- 初始硬编码游戏不再初始化。
