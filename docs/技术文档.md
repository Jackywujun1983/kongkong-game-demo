# 空空如也GameHub 技术文档

## 1. 项目概览

空空如也GameHub 是一个前后端分离源码组织的游戏信息检索网站。当前本地运行时使用一个 Python 标准库 HTTP 服务同时托管静态前端页面和 `/api` 接口，便于原型阶段直接启动和验证。

当前版本已删除账户注册和论坛模块，只保留游戏检索、分类、详情、分页和广告位预留接口。

## 4. 架构说明

```text
浏览器
  └─ HTTP
Python 本地站点服务
  ├─ 托管 frontend/preview.html、detail.html 和静态资源
  └─ 提供 /api REST 接口
      └─ SQLite
```

当前源码仍保持前后端目录分离：

- `backend/` 负责 API、数据库、仓储和服务启动。
- `frontend/` 负责页面和前端工程源码。
- `docs/` 负责需求、技术、数据库、规范和页面布局文档。
- `tools/` 负责数据导入和资源生成。

## 7. API 接口

基础地址：

```text
http://127.0.0.1:8000/api
```

### 7.1 健康检查

```http
GET /api/health
```

### 7.2 分类列表

```http
GET /api/categories
```

### 7.3 游戏检索

```http
GET /api/games?query=银河文明&category=策略SLG&page=1&page_size=48
```

参数：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `query` | string | 否 | 搜索关键词 |
| `category` | string | 否 | 类型 slug 或类型名称，支持多个 |
| `page` | number | 否 | 页码 |
| `page_size` | number | 否 | 每页数量 |

分类筛选规则：

- 精确匹配 `categories.slug` 或 `categories.name`。
- 不使用 `LIKE`。
- 多个类型之间是 OR 关系。

### 7.4 游戏详情

```http
GET /api/games/{slug}
```

### 7.5 广告位预留

```http
GET /api/ads?placement=home_top
```

当前前端不展示广告位。

### 7.6 已移除接口

以下接口不再提供：

- `POST /api/auth/register`
- `POST /api/auth/login`
- `GET /api/me`
- `GET /api/forum/posts`
- `POST /api/forum/posts`
- `GET /api/forum/posts/{id}`
- `POST /api/forum/posts/{id}/comments`

## 10. 测试

后端测试：

```bash
cd backend
python -m unittest discover -s tests
```

当前测试覆盖：

- 游戏搜索。
- 游戏详情。
- 多类型 OR 筛选。
- 分类和广告种子数据。
- 已移除账户/论坛表不再初始化。
- 初始硬编码游戏不再初始化。
