# 空空如也GameHub 数据库设计文档

## 1. 数据库概览

| 项目 | 内容 |
| --- | --- |
| 数据库类型 | SQLite |
| 数据库文件 | `backend/gamehub.sqlite3` |
| 当前用途 | 游戏、分类、多类型关系和广告位预留数据 |
| 已移除模块 | 账户、会话、论坛帖子、论坛评论 |

当前核心表：

| 表名 | 说明 |
| --- | --- |
| `categories` | 游戏类型 |
| `games` | 游戏主数据 |
| `game_categories` | 游戏与类型的多对多关系 |
| `ads` | 广告位预留数据 |

已移除表：

| 表名 | 原用途 | 当前状态 |
| --- | --- | --- |
| `users` | 用户账户 | 已删除 |
| `sessions` | 登录会话 | 已删除 |
| `forum_posts` | 论坛帖子 | 已删除 |
| `forum_comments` | 论坛评论 | 已删除 |

## 2. 表结构

### 2.1 categories

游戏类型表。

| 字段 | 类型 | 约束 | 中文描述 |
| --- | --- | --- | --- |
| `id` | INTEGER | PK | 类型主键 |
| `name` | TEXT | NOT NULL | 类型名称，例如 `动作`、`策略`、`策略SLG` |
| `slug` | TEXT | NOT NULL, UNIQUE | 类型稳定标识，用于 API 分类筛选 |
| `description` | TEXT | NOT NULL | 类型说明 |
| `game_count` | INTEGER | NOT NULL, DEFAULT 0 | 当前关联游戏数量，根据 `game_categories` 统计维护 |
| `is_visible` | INTEGER | NOT NULL, DEFAULT 1 | 是否在分类筛选中展示，`1` 表示展示，`0` 表示隐藏 |

显示规则：

- 根据 `game_categories` 统计每个类型关联的去重游戏数。
- 统计结果写入 `categories.game_count`。
- 关联游戏数小于 `10` 时，`is_visible = 0`。
- 关联游戏数大于等于 `10` 时，`is_visible = 1`。

### 2.2 games

游戏主表。

| 字段 | 类型 | 约束 | 中文描述 |
| --- | --- | --- | --- |
| `id` | INTEGER | PK | 游戏主键 |
| `title` | TEXT | NOT NULL | 游戏名称 |
| `slug` | TEXT | NOT NULL, UNIQUE | 游戏稳定标识，用于详情页 URL |
| `category_id` | INTEGER | NOT NULL, FK -> `categories.id` | 主类型 ID，兼容字段 |
| `studio` | TEXT | NOT NULL | 工作室或发行方，未知时写入默认值 |
| `release_year` | INTEGER | NOT NULL | 发行年份，无法解析时为 `0` |
| `rating` | REAL | NOT NULL | 评分，当前导入数据默认 `0.0` |
| `cover_url` | TEXT | NOT NULL | 封面地址，缺省使用中性占位图 |
| `download_url` | TEXT | NOT NULL, DEFAULT '' | 游戏下载地址 |
| `summary` | TEXT | NOT NULL | 游戏简介，包含资源分类、游戏类型和资源大小 |
| `details` | TEXT | NOT NULL | 游戏详情，包含资源名称、类型、大小、发布时间和下载地址 |
| `platforms` | TEXT | NOT NULL | JSON 字符串，平台列表 |
| `tags` | TEXT | NOT NULL | JSON 字符串，标签列表 |
| `created_at` | TEXT | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 创建时间 |

### 2.3 game_categories

游戏与类型的多对多关系表。

| 字段 | 类型 | 约束 | 中文描述 |
| --- | --- | --- | --- |
| `game_id` | INTEGER | PK, FK -> `games.id` | 游戏 ID |
| `category_id` | INTEGER | PK, FK -> `categories.id` | 类型 ID |
| `sort_order` | INTEGER | NOT NULL, DEFAULT 0 | 类型展示顺序 |

关系说明：

- 一个游戏可以对应多个类型。
- 一个类型可以对应多个游戏。
- 删除游戏时，关联类型关系自动删除。
- 删除类型时，关联关系自动删除。

### 2.4 ads

广告位预留表。当前前端不展示广告布局，但后端保留查询能力。

| 字段 | 类型 | 约束 | 中文描述 |
| --- | --- | --- | --- |
| `id` | INTEGER | PK | 广告主键 |
| `placement` | TEXT | NOT NULL | 广告位标识，例如 `home_top`、`sidebar` |
| `title` | TEXT | NOT NULL | 广告标题 |
| `description` | TEXT | NOT NULL | 广告描述 |
| `image_url` | TEXT | NOT NULL | 广告图片地址 |
| `target_url` | TEXT | NOT NULL | 点击跳转地址 |
| `is_active` | INTEGER | NOT NULL, DEFAULT 1 | 是否启用 |

当前保留广告位：

| placement | 说明 |
| --- | --- |
| `home_top` | 首页顶部广告位预留 |
| `sidebar` | 详情页侧边栏广告位预留 |

已移除广告位：

| placement | 原用途 | 当前状态 |
| --- | --- | --- |
| `forum_feed` | 论坛信息流广告位 | 已删除 |


## 4. 分类筛选规则

后端 `/api/games` 分类筛选使用 `game_categories` 关系表。

- 支持传 `categories.slug`。
- 支持传 `categories.name`。
- 支持逗号分隔或重复参数传多个类型。
- 多类型之间是 OR 关系。
- 不使用 `LIKE` 做类型匹配。
- `/api/categories` 仅返回 `is_visible = 1` 的类型，用于前端分类筛选展示。
- `/api/categories` 按 `game_count DESC, name ASC` 排序。

示例：

```http
GET /api/games?category=策略,策略SLG&page=1&page_size=48
```

## 5. 迁移要求

`initialize_database()` 会执行轻量迁移：

- 创建当前核心表。
- 补充 `games.download_url` 字段。
- 补充 `game_categories` 关系表。
- 补充 `categories.game_count` 字段。
- 补充 `categories.is_visible` 字段。
- 根据类型关联游戏数刷新分类数量和展示状态。
- 清理旧初始硬编码游戏。
- 删除已移除的账户和论坛表。
- 删除 `forum_feed` 广告位。
