# 前端页面布局文档

更新日期：2026-06-14

## 1. 文档说明

本文档用于描述当前 空空如也GameHub 前端页面的布局区域、区域命名、对应文件和可调整位置。后续沟通页面调整时，优先使用本文档中的页面编号和区域名称。

当前版本已删除账户注册和论坛模块，页面布局只保留游戏库首页和游戏详情页。

## 2. 当前前端入口

| 类型 | 文件或地址 | 说明 |
| --- | --- | --- |
| 首页 | `frontend/preview.html` | 当前游戏库首页，也是本地站点默认首页 |
| 详情页 | `frontend/detail.html` | 点击游戏详情后进入的独立页面 |
| 主样式 | `frontend/public/site.css` | 当前静态页面的统一样式文件 |
| 兜底数据 | `frontend/public/game-data.js` | API 不可用时的本地兜底数据 |
| React 工程 | `frontend/src/` | 正式前端工程源码，目前静态预览页是主要运行入口 |

已删除入口：

| 类型 | 原文件 | 状态 |
| --- | --- | --- |
| 账户页 | `frontend/auth.html` | 已删除 |
| 论坛页 | `frontend/forum.html` | 已删除 |

推荐访问方式：

```text
http://127.0.0.1:8000/
```

## 3. 全局布局

### G1 页面外壳：`.shell`

所有静态页面都使用：

```html
<main class="shell">...</main>
```

作用：

- 控制页面最大宽度：`min(1480px, 100%)`。
- 页面居中：`margin: 0 auto`。
- 页面内边距：桌面端 `18px`，`640px` 以下为 `10px`，`420px` 以下为 `8px`。

### G2 顶部导航：`.topbar`

所有页面顶部统一使用：

```html
<header class="topbar">...</header>
```

首页桌面端布局：

```text
┌──────────────┬──────────────────────────────┬──────────────┐
│ 品牌 空空如也GameHub │ 搜索区                        │               │
└──────────────┴──────────────────────────────┴──────────────┘
```

详情页桌面端布局：

```text
┌──────────────────────┬────────────────────────────────────┐
│ 品牌入口               │ 装饰轨                              │
└──────────────────────┴────────────────────────────────────┘
```

对应区域：

| 区域 | class/id | 页面 | 说明 |
| --- | --- | --- | --- |
| 品牌区 | `.brand`、`.brand-mark` | 全部页面 | 点击回到首页 |
| 搜索区 | `.search-panel`、`#search-form` | 首页 | 检索游戏 |
| 详情顶部品牌栏 | `.detail-topbar` | 详情页 | 品牌入口和右侧装饰轨组成完整顶部行 |
| 详情顶部装饰轨 | `.detail-topbar-ornament` | 详情页 | 仅作为视觉平衡，不承载操作 |

响应式规则：

- 宽度小于 `980px` 时，`.topbar` 变为单列。
- 宽度小于 `640px` 时，首页搜索框按钮上下排列。

## 4. 首页布局：`frontend/preview.html`

首页页面结构：

```text
H0 顶部导航
H1 顶部概览区
   ├─ H1-1 焦点游戏位
   └─ H1-2 统计区
H2 分类筛选区
H3 游戏库工具栏
   ├─ 结果数量
   └─ 顶部分页
H4 游戏列表区
H5 底部分页区
```

### H1 顶部概览区：`.overview`

代码位置：

```html
<section class="overview">
  <article class="hero-panel" id="spotlight"></article>
  <aside class="stat-strip" id="stats"></aside>
</section>
```

桌面端布局：

```text
┌──────────────────────────────────────────────┬────────────────────┐
│ H1-1 焦点游戏位                               │ H1-2 统计区         │
└──────────────────────────────────────────────┴────────────────────┘
```

### H1-1 焦点游戏位：`.hero-panel`、`#spotlight`

用途：

- 展示当前分页结果中的第一款游戏。
- 展示封面、标题、游戏简介摘要和“查看详情”按钮。
- 简介摘要取游戏 `summary` 字段，位于标题下方。
- 不展示评分、年份、平台。

桌面端内部布局：

```text
┌────────────────────┬──────────────────────────────────────┐
│ 游戏标题             │ 封面图                                │
│ 游戏简介摘要          │                                      │
│                    │                                      │
│ 查看详情按钮          │                                      │
└────────────────────┴──────────────────────────────────────┘
```

对应区域：

| 区域 | class/id | 说明 |
| --- | --- | --- |
| 焦点外壳 | `.hero-panel`、`#spotlight` | JS 动态填充 |
| 焦点封面 | `.hero-panel img` | 右侧封面，桌面端固定列宽约 `320px` |
| 焦点内容 | `.hero-body` | 左侧纵向内容区 |
| 标题 | `.hero-body h1` | 大标题 |
| 简介摘要 | `.hero-summary` | 展示游戏 `summary`，限制最大行数 |
| 查看详情 | `.hero-body .button` | 固定在右侧内容区底部 |

按钮固定规则：

```css
.hero-body {
  display: flex;
  flex-direction: column;
}

.hero-body .button {
  margin-top: auto;
}
```

### H1-2 统计区：`.stat-strip`、`#stats`

用途：

- 展示收录游戏数量。
- 展示 H1-1 当前焦点游戏的全部类型。
- 展示当前可见结果中的最大年份。
- 展示 H1-1 当前焦点游戏的大小。

类型展示规则：

- “类型”展示 H1-1 当前焦点游戏拥有的全部类型。
- 点击 H4 游戏卡片切换焦点游戏后同步更新。

大小展示规则：

- 展示 H1-1 当前焦点游戏的 `size`。
- `size` 为空时展示“未标注”。
- H1-2 内各统计项使用均匀纵向分布：收录游戏靠近顶部，大小靠近底部，中间项保持一致间距。

对应区域：

| 区域 | class/id | 说明 |
| --- | --- | --- |
| 统计外壳 | `.stat-strip`、`#stats` | JS 动态填充，纵向紧凑列表 |
| 单个统计项 | `.stat-item` | 每个统计行，标签和数值横向排列 |

统计字段顺序：

```text
收录游戏
类型
年份
大小
```

### H2 分类筛选区：`.category-filter-panel`、`#categories`

用途：

- 展示“全部游戏”和数据库中的游戏类型。
- 点击分类后清空顶部搜索框，并调用 `/api/games?category=...`。
- 分类按钮使用网格自动换行布局，不使用横向滚动条。
- 默认展示前 `12` 个类型，超过时通过“展开全部类型 / 收起类型”按钮控制。
- 长类型名单行省略，完整名称保存在 `title` 属性中。

对应区域：

| 区域 | class/id | 说明 |
| --- | --- | --- |
| 分类外壳 | `.category-filter-panel` | 分类区卡片外壳 |
| 分类标题 | `.category-filter-head` | 标题“类型” |
| 分类容器 | `.category-row`、`#categories` | 分类按钮网格父级 |
| 分类按钮 | `.category-button` | 单个分类 |
| 当前选中分类 | `.category-button.active` | 高亮当前分类 |
| 展开按钮 | `.category-toggle-button` | 展开或收起全部类型 |

### H3 游戏库工具栏：`.content-toolbar`

布局：

```text
┌────────────────────────────────────────────────────────────┐
│ 游戏库                         显示 1-48 / 2973  上一页 1/62 下一页 │
└────────────────────────────────────────────────────────────┘
```

对应区域：

| 区域 | class/id | 说明 |
| --- | --- | --- |
| 工具栏外壳 | `.content-toolbar` | 游戏列表上方工具条 |
| 结果数量 | `#result-count` | 当前显示区间和总数 |
| 顶部分页 | `#pagination-top`、`.pagination-bar--compact` | 主要翻页入口 |

### H4 游戏列表区：`.game-grid`、`#game-grid`

普通游戏卡片结构：

```text
┌──────────────────────────────┐
│ 封面图                        │
├──────────────────────────────┤
│ 游戏名                        │
│ 类型  动作 / 策略 / ...        │
│                              │
│ 查看详情                      │
└──────────────────────────────┘
```

对应区域：

| 区域 | class/id | 说明 |
| --- | --- | --- |
| 卡片外壳 | `.game-card` | 单个游戏卡片；点击卡片本体后将该游戏展示到 H1-1 焦点游戏位 |
| 卡片封面 | `.game-card img` | 固定高度约 `220px` |
| 卡片内容 | `.game-card-body` | 纵向 flex 布局 |
| 游戏标题 | `.game-card h2` | 游戏名 |
| 字段行 | `.game-card-facts` | 类型，使用浅底信息行和左侧小标签胶囊 |
| 查看详情按钮 | `.game-card .button` | 卡片底部按钮 |

说明：

- 游戏标题固定在封面下方，不因为下方字段缺失而下浮。
- 类型字段使用浅底信息行样式：左侧为浅黄色小标签胶囊，右侧为小号加粗内容；类型过长时允许换行，不使用省略号截断。
- H4 普通游戏卡片不展示年份和游戏大小。
- 卡片本体和键盘 `Enter` / `Space` 负责更新 H1-1 焦点游戏位。
- H4 卡片从页面下方点击后，应滚动回 H1 顶部概览区，确保用户能看到 H1-1 焦点游戏位变化。
- 点击卡片内“查看详情”按钮时直接进入详情页，不触发焦点位切换。
- 当前展示到 H1-1 的卡片增加 `.is-spotlight` 状态样式。

### H5 分页区：`.pagination-bar`

首页有两个分页位置：

| 区域 | class/id | 位置 | 说明 |
| --- | --- | --- | --- |
| 顶部分页 | `#pagination-top`、`.pagination-bar--compact` | 游戏库工具栏右侧 | 主要翻页入口 |
| 底部分页 | `#pagination-bottom`、`.pagination-bar` | 游戏列表底部 | 辅助翻页入口 |

交互规则：

- 点击分页按钮后，先等待游戏列表重新渲染。
- 渲染完成后滚动到 `.content-toolbar` 附近，保证用户仍能看到顶部分页行。
- `.content-toolbar` 使用 `scroll-margin-top` 避开 sticky 顶部导航。

## 5. 详情页布局：`frontend/detail.html`

详情页页面结构：

```text
D0 顶部导航
D1 详情主体
   ├─ D1-1 左侧封面
   └─ D1-2 右侧详情信息
D2 相关推荐
```

D0 顶部导航在详情页使用 `.detail-topbar`：

```text
┌──────────────────────┬────────────────────────────────────┐
│ 品牌入口               │ 装饰轨                              │
└──────────────────────┴────────────────────────────────────┘
```

说明：

- 品牌入口可点击回到首页。
- 装饰轨只承担视觉平衡，不放置导航按钮。
- 移动端隐藏装饰轨，顶部只保留品牌入口。

### D1 详情主体：`.detail-layout`

桌面端布局：

```text
┌──────────────────────┬────────────────────────────────────┐
│ D1-1 左侧封面          │ D1-2 右侧详情信息                   │
└──────────────────────┴────────────────────────────────────┘
```

样式：

- 左列宽度：`minmax(300px, 420px)`。
- 右列宽度：自适应。
- 间距：`22px`。
- 宽度小于 `980px` 时变为单列。

### D1-1 左侧封面：`.detail-cover`

| 区域 | class/id | 说明 |
| --- | --- | --- |
| 封面外壳 | `.detail-cover` | 边框、圆角、裁切 |
| 封面图片 | `.detail-cover img` | 最小高度桌面端约 `560px`，移动端约 `360px` |

### D1-2 右侧详情信息：`.detail-panel.detail-copy`

内容顺序：

```text
游戏标题
游戏详情
类型
年份 / 平台 / 大小
下载地址展示区
前往下载
```

对应区域：

| 区域 | class/id | 说明 |
| --- | --- | --- |
| 标题 | `.detail-copy h1` | 游戏名，始终位于右侧信息面板顶部 |
| 主信息区 | `.detail-main-content` | 游戏详情、类型、统计宫格、下载地址展示区；桌面端平均分配剩余纵向间隔 |
| 信息块 | `.detail-info-section` | 类型分组容器；桌面端标签和值同一行，使用信息行卡片样式 |
| 信息标签 | `.detail-info-label` | 信息块标题，使用醒目的描边标签样式 |
| 游戏详情 | `.detail-description` | 优先展示 `summary`，无 `summary` 时回退到 `details`；不显示“游戏详情”标题 |
| 类型 | `.detail-type-list` | 以标签形式展示游戏全部类型 |
| 统计宫格 | `.detail-stat-grid` | 年份、平台、大小；桌面端同一行三列 |
| 单个统计 | `.detail-stat` | 每个统计项，标题和值同一行 |
| 底部下载区 | `.detail-download-actions` | 仅承载“前往下载”按钮，始终靠近 D1-2 底部 |
| 下载地址展示区 | `.download-address` | 仅展示下载地址，不支持鼠标选中复制；桌面端标签和值同一行 |
| 下载地址文本 | `.detail-download-link` | 不可点击、不可选中，长链接自动换行 |
| 按钮行 | `.meta-row` | 下载按钮 |

布局间距规则：

- 游戏标题固定在右侧信息面板顶部，“前往下载”按钮固定在底部。
- 主信息区内游戏详情、类型、统计宫格、下载地址展示区之间平均分配剩余纵向间隔，避免中间出现单块巨大空白。
- 小屏幕下长文本较多的行可回落为上下排列，避免撑破布局。

### D2 相关推荐：`.related-section`

| 区域 | class/id | 说明 |
| --- | --- | --- |
| 推荐外壳 | `.related-section`、`#related-root` | 详情主体下方 |
| 推荐网格 | `.related-grid` | 自动网格 |
| 推荐卡片 | `.related-card` | 同分类游戏卡片 |

推荐卡片信息规则：

- 推荐卡片展示类型、游戏标题、年份、平台、大小。
- 推荐卡片不展示评分。
- 推荐卡片不展示简介摘要。

## 6. 响应式布局规则

### R1 中等屏幕：`max-width: 980px`

触发后：

- `.topbar` 变为单列。
- `.overview` 变为单列，焦点位和统计区上下排列。
- `.detail-layout` 变为单列，详情封面和详情信息上下排列。
- `.hero-panel` 变为单列，焦点图和文字上下排列。
- `.detail-topbar` 保留品牌入口和装饰轨的两列结构。

### R2 小屏幕：`max-width: 640px`

触发后：

- `.shell` 内边距缩小为 `10px`。
- `.search-panel` 变为单列，搜索输入框和按钮上下排列。
- `.detail-topbar` 变为单列，并隐藏 `.detail-topbar-ornament`。
- `.detail-stat-grid` 变为单列。
- `.content-toolbar` 和 `.toolbar-actions` 变为纵向排列。
- 顶部分页 `.pagination-bar--compact` 左对齐。

### R3 极小屏幕：`max-width: 420px`

触发后：

- `.shell` 内边距缩小为 `8px`。
- 首页焦点图、普通卡片图和推荐卡片图高度降低。
- 详情页封面最大高度降低。

## 8. 后续调整建议

- 修改首页布局时，优先确认目标区域属于 `H1 焦点位`、`H3 工具栏` 还是 `H4 普通游戏卡片`。
- 修改按钮固定位置时，优先检查父容器是否使用纵向 flex，以及目标元素是否使用 `margin-top: auto`。
- 修改字段展示时，优先确认字段来自 API 数据、`normalizeGame` 转换逻辑，还是纯 CSS 展示。
- 修改移动端表现时，同时检查 `max-width: 980px` 和 `max-width: 640px` 两组媒体查询。
- 当前账户注册和论坛模块已删除；如需恢复，应先更新需求文档、页面布局文档、接口文档和数据库设计。
