# 空空如也GameHub 游戏信息检索网站需求文档

更新日期：2026-06-14

## 1. 项目定位

空空如也GameHub 是一个面向玩家的游戏信息检索网站，当前聚焦游戏搜索、类型筛选、游戏库分页、游戏详情、下载地址展示和同分类推荐。

当前版本不包含账户注册、登录权限和论坛功能。广告位数据能力在后端保留为预留接口，前端正式页面不展示广告布局。

## 2. 当前功能范围

### 2.1 游戏检索

- 用户在首页顶部搜索框输入关键词并点击“检索”。
- 后端检索接口为 `GET /api/games`。
- 检索字段包括游戏名称、简介、标签和工作室。
- 搜索结果支持分页。
- GitHub Pages 静态版无法访问 `/api` 时，前端自动使用 `frontend/public/game-data.js` 兜底数据。

### 2.2 游戏类型筛选

- 首页展示来自 `categories` 表的可见游戏类型。
- `/api/categories` 仅返回 `is_visible = 1` 的类型。
- 当前可见规则：关联游戏数大于等于 `10` 的类型展示在筛选区。
- 点击类型按钮时，必须清空顶部搜索框内容，只按当前类型筛选。
- 一个游戏可以对应多个类型。
- 类型筛选命中规则：只要游戏拥有的任意一个类型满足条件，就返回该游戏。
- 后端类型匹配使用 `categories.slug` 或 `categories.name` 精确匹配，不使用 `LIKE`。
- 多个类型筛选值之间是 OR 关系。

### 2.3 首页焦点游戏位

首页 H1-1 焦点游戏位用于展示当前分页结果中的重点游戏。默认展示当前结果页第一款游戏；点击 H4 游戏卡片本体后，会切换为被点击游戏并滚动回顶部概览区。

焦点游戏位展示：

- 游戏封面。
- 游戏名称。
- 游戏简介摘要，来源于 `summary`。
- “查看详情”按钮。

焦点游戏位不展示：

- 类型行。
- 评分。
- 年份。
- 平台摘要。

### 2.4 首页统计区

首页 H1-2 统计区展示当前游戏库状态：

| 统计项 | 数据口径 |
| --- | --- |
| 收录游戏 | 当前筛选结果总数 |
| 类型 | H1-1 当前焦点游戏拥有的全部类型 |
| 年份 | 当前可见结果中的最大年份 |
| 大小 | H1-1 当前焦点游戏的 `size`，为空时展示“未标注” |

### 2.5 游戏列表

首页 H4 游戏列表卡片展示：

- 游戏封面。
- 游戏名称。
- 类型信息行。
- “查看详情”按钮。

展示约束：

- 游戏名称始终固定在封面下方。
- 类型信息行使用浅底信息块，左侧是“类型”标签，右侧展示游戏全部类型。
- 类型文本过长时允许换行，不使用省略号截断。
- H4 普通游戏卡片不展示年份。
- H4 普通游戏卡片不展示游戏大小。
- “查看详情”按钮固定在卡片底部。
- 点击卡片本体切换 H1-1 焦点游戏位。
- 点击“查看详情”按钮进入独立详情页，不触发焦点位切换。
- 封面为空或加载失败时，统一显示默认封面 `frontend/public/assets/covers/default-game-cover.jpg`。

### 2.6 游戏详情

点击“查看详情”进入 `frontend/detail.html?slug=...` 独立详情页。

详情页展示：

- 顶部品牌入口。
- 游戏封面。
- 游戏名称。
- 游戏简介，优先取 `summary`，无值时回退到 `details`。
- 游戏全部类型。
- 发行年份。
- 支持平台。
- 游戏大小。
- 下载地址展示区。
- “前往下载”按钮。
- 同分类推荐。

详情页顶部规则：

- 正常详情页不展示“游戏库”按钮。
- 正常详情页不展示“返回游戏库”按钮。
- 正常详情页不展示面包屑导航。
- 顶部品牌区是返回首页入口，需要具备可点击感和鼠标悬浮反馈。
- 右侧装饰轨只用于视觉平衡，不承载业务操作。

下载地址规则：

- 下载地址文本不做直接链接。
- 下载地址文本不可被鼠标选中复制。
- “前往下载”按钮在有下载地址时打开下载地址。

同分类推荐规则：

- 推荐数据来自同类型游戏。
- 推荐卡片展示类型、游戏名称、年份、平台和大小。
- 推荐卡片不展示简介摘要。

### 2.7 分页

首页支持分页浏览全部游戏数据。

分页位置：

- H3 游戏库工具栏右侧提供顶部分页。
- H5 游戏列表底部保留底部分页。

交互规则：

- 点击上一页、下一页、首页、尾页后重新加载当前页数据。
- 翻页后页面聚焦位置保持在可以看到顶部分页行的位置。
- 前端当前每页展示 `48` 条，后端 `page_size` 最大限制为 `50`。

### 2.8 广告位预留

当前前端不展示广告位。

后端保留广告位接口：

```text
GET /api/ads?placement=
```

当前保留广告位：

- `home_top`
- `sidebar`

## 3. 已移除功能

以下模块已从当前版本移除：

- 账户注册。
- 用户登录。
- 登录会话。
- 权限校验。
- 论坛帖子列表。
- 论坛发帖。
- 论坛评论。
- 论坛信息流广告位。

后续如需恢复，应作为新需求重新设计接口、页面、数据库表和权限规则。

## 4. 用户角色

当前版本不区分登录用户和游客，所有访问者均可：

- 浏览游戏列表。
- 搜索游戏。
- 使用类型筛选。
- 查看游戏详情。
- 通过“前往下载”打开下载地址。

## 6. 接口范围

| 接口 | 方法 | 说明 |
| --- | --- | --- |
| `/api/health` | GET | 健康检查 |
| `/api/categories` | GET | 获取可见游戏类型 |
| `/api/games` | GET | 游戏列表、搜索、类型筛选、分页 |
| `/api/games/{slug}` | GET | 游戏详情 |
| `/api/ads` | GET | 广告位预留接口 |

已移除接口：

- `/api/auth/register`
- `/api/auth/login`
- `/api/me`
- `/api/forum/posts`
- `/api/forum/posts/{id}`
- `/api/forum/posts/{id}/comments`

## 7. 数据要求

当前核心数据来自 `data/fufu_quark_only.csv` 导入结果。

游戏数据要求：

- `games` 保存游戏主信息。
- `categories` 保存去重后的游戏类型。
- `game_categories` 保存游戏和类型的多对多关系。
- `games.download_url` 保存游戏下载地址。
- `games.size` 保存游戏大小，导入时由 CSV 的 `大小` 字段写入。
- `games.cover_url` 保存封面地址，缺失或加载失败时前端统一使用默认封面。
- 静态部署前应通过 `tools/export_static_game_data.py` 将 SQLite 数据导出到 `frontend/public/game-data.js`。

## 8. 验收标准

- 首页可以展示数据库导入后的游戏列表。
- 搜索功能可以返回匹配游戏。
- 点击类型筛选会清空搜索框，并只按类型筛选。
- 类型筛选基于多类型关系，任意类型命中即可返回。
- 分页可以访问全部游戏。
- 点击 H4 卡片本体可以切换 H1-1 焦点游戏位，并滚动到顶部概览区。
- 点击“查看详情”可以跳转到独立详情页。
- 游戏详情页可以展示下载地址和“前往下载”按钮。
- 游戏详情页正常状态不展示“游戏库”和“返回游戏库”按钮。
- 详情页品牌区可点击返回首页，并具备 hover 动态反馈。
- 首页和详情页使用统一默认封面兜底。
- 首页不展示账户注册入口。
- 首页不展示论坛入口。
- 后端不再创建账户、会话、论坛帖子、论坛评论表。
