# 游戏类型枚举文档

生成时间：2026-06-14

## 1. 文档说明

本文档根据当前 SQLite 数据库中的 `categories` 表生成，用于统一游戏类型枚举值。前端类型筛选、后端查询、数据导入和后续需求沟通，应优先使用本文档中的类型名称与 slug。

## 2. 数据来源

| 项目 | 内容 |
| --- | --- |
| 数据库 | `backend/gamehub.sqlite3` |
| 类型表 | `categories` |
| 类型关联表 | `game_categories` |
| 去重字段 | `categories.name` |
| API 推荐传值 | `categories.slug` 或 `categories.name` |
| 当前游戏总数 | 2973 |
| 当前去重类型数 | 49 |
| 当前显示类型数 | 35 |
| 当前隐藏类型数 | 14 |
| 当前类型关联数 | 8546 |

统计 SQL：

```sql
SELECT
    categories.id,
    categories.name,
    categories.slug,
    categories.is_visible,
    categories.game_count
FROM categories
ORDER BY categories.name COLLATE NOCASE;
```

## 3. 使用约定

- `类型名称` 是页面展示和业务沟通使用的中文或英文类型值。
- `slug` 是当前数据库中用于 API 类型筛选的稳定标识。
- `is_visible` 表示是否在首页类型筛选区展示，`1` 表示展示，`0` 表示隐藏。
- 关联游戏数小于 `10` 的类型会隐藏，大于等于 `10` 的类型会展示。
- 前端请求 `/api/games` 时，推荐优先传 `slug`。
- `/api/categories` 仅返回 `is_visible = 1` 的类型。
- `/api/categories` 按 `game_count DESC, name ASC` 排序。
- 后端支持传入单个或多个类型，多个类型之间是 OR 关系。
- 本文档按当前数据库实际值生成，不额外过滤低频类型。
- 如果后续清理、合并或新增类型，需要同步更新本文档。

## 6. 前端筛选示例

按 slug 查询：

```http
GET /api/games?category=fufu-category-f3c49831c636&page=1&page_size=48
```

按类型名称查询：

```http
GET /api/games?category=策略&page=1&page_size=48
```

按仙侠类型查询：

```http
GET /api/games?category=fufu-category-aef641692c65&page=1&page_size=48
```

多类型 OR 查询：

```http
GET /api/games?category=策略,仙侠&page=1&page_size=48
```

说明：多类型筛选是 OR 关系，只要一个游戏关联的任意类型命中，就会返回该游戏。
