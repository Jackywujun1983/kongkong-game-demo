# 空空如也GameHub 前端 UI 设计规范

## 1. 文档定位

本文档记录当前项目最终确认的前端 UI 设计方向，作为后续页面优化、样式调整、组件重构和移动端适配的执行依据。

后续涉及以下文件的 UI 改动，应优先遵守本文档：

- `frontend/preview.html`
- `frontend/detail.html`
- `frontend/public/site.css`
- `frontend/src/styles/global.css`
- `frontend/src/features/games/`

如未来需要调整整体视觉方向，应先更新本文档，再执行代码修改。

## 2. 设计关键词

当前 UI 方向确定为：

- 素雅
- 游戏动感
- 纸面质感
- 粗描边
- 低饱和色彩
- 信息清晰
- 移动端友好

避免方向：

- 高饱和霓虹风
- 暗黑赛博风
- 横向强制滚动筛选
- 过重渐变背景
- 大面积单一蓝紫色
- 按钮和卡片因数据缺失产生跳动

## 3. 视觉语言

### 3.1 页面基调

页面采用浅色纸面背景，配合细网格纹理，整体应像“游戏资料档案馆”，而不是传统暗色游戏站。

核心视觉特征：

- 背景为浅米白色。
- 卡片为白色或浅绿灰色。
- 边框使用深色粗描边。
- 阴影使用偏硬朗的低透明投影。
- 圆角保持克制，统一为 `8px`。

### 3.2 主色板

当前色彩以 `frontend/public/site.css` 中变量为准：

| 用途 | 变量 | 当前值 | 说明 |
| --- | --- | --- | --- |
| 页面背景 | `--bg` | `#f6f7ef` | 浅米白纸面 |
| 网格背景 | `--bg-grid` | `rgba(32, 34, 31, 0.035)` | 低对比网格 |
| 主卡片 | `--surface` | `#ffffff` | 内容底色 |
| 柔和卡片 | `--surface-soft` | `#eef3e8` | 数据块、标签底色 |
| 深色块 | `--surface-strong` | `#20221f` | 统计强调块 |
| 文字 | `--text` | `#20221f` | 主文本 |
| 辅助文字 | `--muted` | `#697268` | 次级信息 |
| 主按钮 | `--primary` | `#bfd8c6` | 检索、主操作按钮 |
| 主强调文字 | `--primary-strong` | `#5d7f6a` | 类型、标签文字 |
| 详情按钮 | `--info` | `#c7d0ad` | 查看详情按钮 |
| 选中态 | `--warning` | `#eccb66` | 当前导航、选中类型 |
| 风险/下载地址 | `--danger` | `#f16a53` | 下载地址文本 |

### 3.3 默认封面

默认封面文件：

- `frontend/public/assets/covers/default-game-cover.jpg`
- `frontend/public/assets/covers/default-game-cover.png`

设计要求：

- 3:4 竖版游戏封面比例。
- 素雅浅色底。
- 具备轻微游戏动感，如光环、速度线、网格透视、空间纵深。
- 不出现文字、Logo、人物、武器、水印。
- 不使用高对比暗色霓虹风。
- 首屏加载优先使用 JPG。

## 4. 页面结构

### 4.1 全局容器

所有静态页面使用：

```html
<main class="shell">...</main>
```

约束：

- 桌面端最大宽度为 `1480px`。
- 页面居中。
- 桌面端保留舒展留白。
- 手机端压缩边距，优先展示内容。

### 4.2 顶部导航

顶部导航使用 `.topbar`，包含：

- 品牌区 `.brand`
- 首页搜索区 `.search-panel`

设计规则：

- 顶部区域保持 sticky。
- 搜索框是首页核心操作，按钮文案为“检索”。
- 检索按钮使用淡鼠尾草绿，不使用强蓝或强青色。
- 首页和详情页顶部不展示“游戏库”导航按钮。
- 移动端顶部区域允许纵向堆叠。
- 移动端搜索按钮占满搜索框宽度。

## 5. 首页 UI 规范

### 5.1 焦点游戏区

区域：

- `.overview`
- `.hero-panel`
- `.hero-body`
- `.stat-strip`

布局：

- 桌面端左侧展示焦点游戏信息，右侧展示默认封面。
- 右侧统计区展示收录游戏、类型游戏、最高评分、最新年份。
- 移动端焦点图高度降低，避免首屏被大图占满。

内容规则：

- 焦点游戏标题可大字号展示。
- 不展示简介摘要。
- 不展示评分、年份、平台摘要行。
- “查看详情”按钮固定在焦点区底部。

### 5.2 类型筛选区

区域：

- `.category-filter-panel`
- `.category-filter-head`
- `.category-row`
- `.category-button`
- `.category-toggle-button`

设计规则：

- 不使用横向滚动条。
- 不使用单行横向滑动选择。
- 类型按钮使用自动换行网格。
- 不展示“点击类型筛选游戏”等辅助说明文案。
- 默认展示前 `12` 个类型。
- 类型超过默认数量时，显示底部按钮：
  - `展开全部类型（数量）`
  - `收起类型`
- 展开按钮独立占整行，不和类型按钮挤在同一个小格子里。
- 类型按钮文字默认不换行。
- 类型按钮宽度应跟随网格自适应，避免短按钮和换行按钮混排导致视觉松散。
- 长类型名使用省略号，并通过 `title` 保留完整名称。
- 选中类型使用柔和黄色背景。

交互规则：

- 点击类型时应清空顶部搜索框内容。
- 点击类型后只按照当前游戏类型进行筛选，不再叠加上一次搜索关键词。
- 一个游戏有多个类型时，任意一个类型匹配即命中。

### 5.3 游戏列表区

区域：

- `.content-toolbar`
- `.game-grid`
- `.game-card`
- `.game-card-body`
- `.game-card-facts`
- `.game-card-tags`

卡片内容：

- 封面图。
- 游戏名称。
- 类型。
- 年份，缺失则不展示该行。
- 底部胶囊只展示游戏大小。
- 游戏大小缺失时，底部胶囊整行不展示。
- 查看详情按钮。

关键约束：

- `查看详情` 按钮必须固定在卡片底部。
- 按钮不能因为年份、大小等数据缺失而上浮。
- 卡片标题最多显示两行。
- 默认封面加载失败时统一回退到 `default-game-cover.jpg`。

### 5.4 分页

分页区域：

- 顶部分页：`.pagination-bar--compact`
- 底部分页：`.pagination-bar`

规则：

- 上一页、下一页操作后，页面聚焦位置应保持在能看到分页行的位置。
- 移动端分页按钮采用两列弹性布局。
- 禁用按钮降低透明度，但保持位置稳定。

## 6. 详情页 UI 规范

### 6.1 详情页顶部

规则：

- 详情页正常状态不展示“游戏库”按钮。
- 详情页正常状态不展示“返回游戏库”按钮。
- 详情页正常状态不展示面包屑导航。
- 顶部使用 `.detail-topbar` 专用品牌栏。
- 左侧品牌区作为可点击的首页入口。
- 右侧使用低调装饰轨撑起整行视觉，不放置额外导航按钮。
- 品牌区 hover 时应有轻微位移、投影或颜色变化，让用户感知可点击。
- 移动端隐藏右侧装饰轨，保留品牌入口。
- 未找到游戏等兜底状态可以保留返回入口。

### 6.2 详情主体

区域：

- `.detail-layout`
- `.detail-cover`
- `.detail-panel.detail-copy`

内容规则：

- 左侧为封面。
- 右侧为类型、标题、下载地址、评分、发行年份、游戏大小、平台和下载按钮。
- 下载地址只展示文本，不直接点击，不允许选中复制。
- 工作室位置已调整为游戏大小。
- 不展示标签胶囊。

移动端规则：

- 封面限宽限高。
- 信息区尽快出现在首屏下方。
- 数据块变为单列，避免挤压。

### 6.3 同分类推荐

区域：

- `.related-section`
- `.related-grid`
- `.related-card`
- `.related-card-body`
- `.related-stat-grid`

展示规则：

- 同分类推荐的数据口径必须和详情页主体一致。
- 推荐卡片展示：
  - 类型
  - 游戏标题
  - 评分
  - 发行年份
  - 游戏大小
  - 支持平台
- 不再展示简介摘要。
- 移动端推荐卡片单列展示。

## 7. 按钮规范

### 7.1 检索按钮

用途：

- 首页搜索提交。

样式：

- 使用 `--primary`。
- 文字使用深色 `--text`。
- 不使用高饱和蓝色或青色。

### 7.2 查看详情按钮

用途：

- 游戏卡片进入详情页。

样式：

- 使用 `--info`。
- 文字使用深色 `--text`。
- 在卡片中使用 `margin-top: auto` 固定到底部。

### 7.3 展开全部类型按钮

用途：

- 展开或收起类型筛选面板。

样式：

- 独立占整行。
- 居中显示。
- 单行文本，不换行。
- 使用柔和浅色背景和低对比边框。

## 8. 移动端专项规范

必须支持以下宽度：

- `390px`
- `360px`
- `320px`

验收要求：

- 页面无横向滚动。
- 顶部搜索可完整显示。
- 首页焦点图不能占满首屏。
- 游戏卡片按钮固定底部。
- 类型筛选默认不铺满过长页面。
- 详情页封面不超过合理高度。
- 同分类推荐单列展示。

当前移动端策略：

- `.topbar` 单列堆叠。
- `.search-panel` 搜索框和按钮上下排列。
- `.hero-panel` 单列，封面降低高度。
- `.game-grid` 单列。
- `.detail-layout` 单列。
- `.related-grid` 单列。
- `.pagination-button` 两列弹性排列。

## 9. 后续优化执行原则

后续 UI 优化必须遵守：

1. 先确认是否影响本文档定义的设计方向。
2. 不重新引入横向类型滑动筛选。
3. 不重新引入账户注册和论坛入口，除非需求文档先变更。
4. 不展示前端广告位，除非需求文档先变更。
5. 不将按钮改回高饱和蓝色、青色或霓虹色。
6. 不让卡片按钮因数据缺失产生位置跳动。
7. 不让下载地址变成可点击或可复制。
8. 修改静态页时，同步考虑 React 源码中的对应组件。
9. 修改布局后至少验证桌面端和 `390px`、`320px` 移动端。

## 10. 实现文件映射

| UI 区域 | 主要文件 | 说明 |
| --- | --- | --- |
| 首页静态页 | `frontend/preview.html` | 搜索、焦点区、类型筛选、游戏列表、分页 |
| 详情静态页 | `frontend/detail.html` | 返回入口、详情主体、下载地址、同分类推荐 |

| React 全局样式 | `frontend/src/styles/global.css` | 与静态页关键视觉规则保持一致 |

## 11. 验证清单

每次重要 UI 修改后，至少检查：

- 首页是否正常加载游戏列表。
- 首页类型筛选是否可点击。
- 类型展开/收起是否正常。
- 搜索后再切换类型是否清空搜索框，并只按类型筛选。
- 游戏卡片 `查看详情` 是否固定底部。
- 详情页是否能打开。
- 下载地址是否只展示文本。
- 同分类推荐是否展示详情页一致数据。
- `390px` 和 `320px` 宽度下是否无横向滚动。

## 12. 当前设计状态

本文档所描述的 UI 设计为当前项目最终确认的执行方向。

后续前端页面优化、样式调整、交互调整，应以本文档为基准。
